Here's how it's organized:

**Timeline structure** — The 20 weeks are divided into 6 color-coded phases, each mapped to a month. Python was moved to Week 5–6 (Phase 2) so you build scripting skills before Jenkins and Ansible, which makes both of those courses significantly easier.

**Kubernetes gets 5 weeks (Weeks 11–15)** — twice the time of any other course. This reflects its real-world complexity and the fact that it's the highest-demand skill in the list. Rushing it would be a mistake.

**Each course has 3–4 🔑 Critical Topics**, each with a detailed explanation of _why_ it matters, _what_ specifically to study within it, and code/config examples you can reference directly. Every course also ends with a hands-on practice project.

**The daily 6-hour block** is broken into: 2 hrs theory → 2 hrs hands-on labs → 1 hr documentation → 1 hr mini-projects/review. This ratio is intentional — passive watching without immediate practice is the #1 reason people fail to retain DevOps skills.

**The Capstone (Week 20)** ties every tool together into a single end-to-end pipeline: code push → Jenkins CI → Docker image → Kubernetes rolling deploy — all on infrastructure provisioned by Terraform and configured by Ansible.

The roadmap also closes with a **Common Mistakes** section and a **Supplemental Resources** table so you always know where to go deeper on any topic.
